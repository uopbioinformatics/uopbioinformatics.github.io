Introduction to Bioinformatics

Abstract

An introductory lecture giving a brief outline of what bioinformatics entails, the role of bioinformatics in modern biological sciences research, and the tools that a bioinformatician must learn to analyse the large body of data generated from such research. This will be a 2 hour introductory session geared towards MSc students in the School of Pharmacy and Biomedical Sciences.

Date
Tue, Nov 7, 2017
Event
MSc Medical Biotechnology
Location
Portsmouth, United Kingdom

Introduction

The term “Bioinformatics” can apply to a wide range of disciplines in the biological sciences, and the field lies at the interface between biology, statistics, mathematics and computer science. The following are suggested training modules that represent what I believe to be the key aspects of Bioinformatics knowledge essential for anyone looking to analyse next generation sequencing (NGS) data. In this first lecture, we will discuss the following topics in brief to gain an understanding of what Bioinformatics actually is and how essential it is becoming as Big Data becomes the norm in the Biological Sciences.

Bioinformatics Essentials

This series of modules aims to cover some of the basics of Bioinformatics.

1) What is Bioinformatics?

This course will introduce the basic concepts behind Bioinformatics analyses, and the reasons why Bioinformaticians are becoming more and more essential to cope with the influx of data as technology continues to improve. We will discuss some examples of how computational methods can help to solve biological problems, and the sorts of typical analyses that a Bioinformatician may expect to conduct.

2) Sequence Alignment

This course will focus on the idea of sequence alignments, aiming to find the “best” alignment between two sequences. In Bioinformatics, we often want to identify how similar two sequences are, be these sequences of nucleotides in DNA and RNA, or sequences of amino acids in proteins. The concept of a “best” alignment will be discussed, as often how we interpret this will be dependent on the type of data and the questions that we are hoping to answer.

We will also discuss some of the biology behind these questions, why we might be interested in the similarity between two sequences, and how these may relate to the evolution of functional elements between species.

We will discuss some of the algorithms available for performing alignments (e.g. dynamic programming methods such as Smith-Waterman alignments, or heuristic k-tuple methods such as those utilised by BLAST and BLAT) and the various parameters that need to be considered for such analyses (e.g. gap penalties, substitution matrix selection, etc.).

We will touch on the FASTA format of representing sequences, since these are typically the format of choice for most alignment tools, and will discuss some of the more well known sequence alignment tools such as ClustalW for alignment of multiple sequences (e.g. in homology searches) and BLAST for sequence pattern searches in protein and nucleotide databases (such as Ensembl).

We will also discuss how this leads into motif analyses; searching for common short patterns shared between multiple sequences. Such motifs may represent common DNA binding sites (e.g. transcription factor binding sites), or functional protein domains with shared functions.

Finally, this will lead into the idea of pattern matching of short sequences against a complete genome, which is the basis of NGS.

3) Interrogation of Bioinformatics Resources

Often Bioinformaticians will need to access published data or data curated by some external resource to compare against their own data. In this module we will discuss some of the more common databases (GenBank, RefSeq, UCSC, Uniprot, Ensembl, ArrayExpress, GEO, SRA, Pfam, Interpro, MiRBase, KEGG, etc). This course will discuss the role of some of these databases, how they are maintained and who maintains them, what data can be accessed from them, and how to access the data. We will also discuss how to submit data to curated databases such as ArrayExpress and GEO, which is typically a requirement of publication of any kind of sequencing based project.

4) The Genome Browser

This course will look at how to visualise genomic data using genome browsers such as the University of California, Santa Cruz (UCSC) genome browser or the Integrative Genomics Viewer (IGV). The concepts of data tracks will be discussed, including coverage tracks and bed tracks. We will discuss some of the tracks available in the UCSC browser including annotation from various databases, conservation, and links to the incredibly useful ENCODE (Encylopedia of DNA Elements) data set. We will also look at how we can upload our own tracks, how we can use the table browser to access data, how to convert between different genome builds using the liftOver tool, and how we can set up and share sessions.

Next Generation Sequencing

The following modules aim to study the theories underlying NGS analyses, the methods that are available to researchers and how they can be used, and the tools available for the analysis of these data.

1) Introduction

This module will aim to introduce the concept of NGS, and the broad range of analyses that it opens up to researchers. The idea behind NGS is to utilise the power of modern sequencing chemistries to interrogate a large number of short nucleotide fragments simultaneously for global analyses. We will discuss the evolution of sequencing from the Sanger sequencing techniques used to build the first human genome, to the current Illumina sequencing-by-synthesis approach that allows parallel reading of millions of sequence fragments in a single run.

We will discuss how the current sequencing technologies work (focusing primarily on Illumina, but also discussing 454 pyrosequencing, SOLiD, Ion Torrent, etc.), and some of the key considerations for the generation of libraries to be used in sequencing reactions. We will focus mostly on Illumina technologies and will discuss the library preparation methods, how bridge amplification works to generate clusters on the flow cell, the cyclic nature of fluorescently labeled nucleotide incorporation to generate sequence reads, and how base calling scores are generated by the machine.

The idea of NGS is to either resequence an unknown genome to try and build contigs into a complete scaffold, or else to map short fragments against a known genome build in order to quantify different loci (e.g. regions of DNA bound by a transcription factor, or expressed genes). We will discuss the idea of read “mapping”, and the different algorithms available for this.

In particular we will focus on two main NGS applications:

  1. ChIP-seq: Sequencing of DNA fragments identified through immunoprecipitation pulldown, typically used to identify protein-DNA interactions (e.g. transcription factor binding) or DNA bound to histones showing specific modifications (e.g. H3K4me3)
  2. RNA-seq: Sequencing of RNA fragments in order to profile gene (and other non-coding transcript) expression

We will discuss the key differences between these two types of data (in particular the difference between mapping to the genome and mapping to the transcriptome), and what this means for mapping. We will contrast genomic mapping algorithms (e.g. BWA and Bowtie) with splice-aware mapping algorithms (e.g. Tophat2 and STAR).

We will briefly discuss the broad range of additional sequencing approaches available to researchers (RIP-seq, CLIP-seq, ChIRP-seq, Bisulphite-seq, NET-seq, etc.), and the situations in which these might be of use.

2) Experimental Design

This module will focus more on the types of considerations that must be undertaken by a Bioinformatician when designing an NGS experiment. Along with more basic elements of experimental design such as power calculations, determining the minimum number of replicates, blocking and randomisation to remove systematic bias, etc, we will focus on how one goes about selecting the optimum library design for the question of interest. This includes whether or not paired end sequencing should be considered, the number of sequencing cycles to use, whether to use multiplexing, the read depth required and the number of lanes required to achieve this, whether a stranded protocol should be utilised, etc.

In addition, certain NGS specific issues will be discussed such as selection of multiplex barcodes to ensure sequence complexity is maintained, balancing of multiplexed samples to avoid biasing a sequencing run for a single sample, how best to mutliplex samples over multiple lanes, adapter contamination and how to avoid it, PCR bias and how to avoid it, selection of control samples (e.g. input controls and IgG negative controls), etc.

We will discuss how best to avoid typical pitfalls of NGS experiments, and how to deal with issues when they do arise. We will also discuss some of the issues inherent to different sequencing technologies, such as how best to account for ribosomal contamination in total RNA when looking at RNA seq (ribozero treatment, polyA selection, or pre-mapping against a ribosomal transcriptome), or how to deal with cytosine-depletion in bisulphite treated sequencing samples.

3) Mapping

This module will focus on what to do with the raw data that comes from an NGS experiment. We will discuss the fastq file format, and how the base quality scoring is represented in Phred format. We will touch on the fact that the encoding for these can be different depending on when and where the sequencing was performed.

We will discuss the initial quality control steps that should be performed to determine the quality of these data prior to mapping. For instance we can check that the reads do indeed come from the intended species with no contamination, we can check for the presence of adapter contamination (or any other sequence bias) that may represent a library problem that needs to be resolved prior to mapping, we can check that base composition is roughly uniform across the read set, we can check for read duplication which may represent PCR bias in the library preparation stage, etc.

We will discuss the idea of read trimming and some of the available tools for this (trim-galore, trimmomatic, fastx toolkit), to remove low-quality bases and contaminating sequences which may prevent accurate mapping.

We will discuss how to go about choosing the best mapping algorithm for your needs and how to tune the parameters to get the most accurate mapping from your data. The idea of mapping quality will be discussed, and how this is determined. In particular we will discuss issues that can cause ambiguity in mapping. For instance we may see many reads mapping to a single location, which may represent a result of PCR amplification bias. Or we may see a single read map to many locations (multi-reads), typically representing duplicated regions of the genome (e.g. segmental duplications, repetitive elements and the like). There is currently no consensus on how best to deal with these ambiguities, but knowing the options is essential.

We will talk about the current standardised format for mapped data (SAM and BAM format), and how to understand the meaning of the different fields. We will use the samtools suite of tools for interrogating, sorting, filtering, etc. these files, as well as additional tools for generating formats for use with genome browsers such as UCSC and IGV (e.g. the bedtools suite which can convert SAM/BAM format files into genome coverage files such as bedgraph or wiggle files).

4) Next Generation Sequencing Analysis

This module will focus on what to do with our mapped data, and will focus primarily on the analysis of ChIP-seq and RNA-seq data. For ChIP-seq, the idea is to identify loci that are enriched for the factor that we are interrogating. For many factors (e.g. transcription factors) this comes down to identifying “peaks” in the data that are significantly enriched above the background. Various peak calling algorithms are available for this, and we will discuss some of these (e.g. MACS, FindPeaks, SICER, etc.) and how they work. Some factors (e.g. histone modifications such as H3K36me3) are not represented by clearly defined peaks, but rather by more protracted regions of enrichment, and so different approaches may need to be considered for these. We will discuss the kinds of analyses typical to ChIP-seq, such as annotating the location of peaks relative to known features (e.g. gene TSS, promoters, exons, introns, enhancers, superenhancers, etc.), comparing profile plots centered at particular regions (e.g. centering all samples at the peaks called for one of the samples to look for correlation, or centering at the TSS to look at promoter association), or identifying differentially regulated regions between conditions.

For RNA-seq, the approach is very different, as we aim to use the mapped reads to generate a quantitative count for known transcripts in order to compare and contrast expression levels between conditions. The idea is to count up reads within the coding region of known genes, although this brings up a number of issues that need to be considered. To avoid biasing our data, we need to ensure that each read is assigned to only a single transcript, but often genes have multiple transcipts consisting of different combinations of exons. Determining which of the transcripts a particular read belongs can be an incredibly tricky endeavor. Also, we see overlapping transcripts in the genome, or features within features (e.g. ncRNAs in introns) which increase this ambiguity. It is very important to be aware of these issues when trying to quantify your transcripts. Typically analyses utilise these per-transcript or per-gene counts and look for differential expression between conditions to find significantly changing candidates. These genes can then be further analysed by looking for common functions (e.g. through the use of gene ontology analyses), and by comparing between different conditions (e.g. different time points) to determine a network of gene expression changes.

Programming

1) Thinking like a programmer

The concept of writing computer code can be a daunting prospect, particularly for people who have had no experience with the basic concepts behind seemingly complex code syntax. This module will be a basic course, introducing people with no computing experience to the concepts of writing programs, mainly through the use of pseudo-code.

Like spoken languages, the concepts underlying most programming languages are very similar. This course will focus on the underlying “grammar” of programming, and not on any one specific language. This includes understanding concepts such as variables, data classes, subroutines/functions, input/output, iteration using loops, conditional statements, boolean operators, comparison operators, basic logic, and best practices for writing code.

With these tools, students will be much better equipped to understand how to answer biological questions using programmatic means.

2) Data processing with perl

In Bioinformatics, we deal with large amounts of data and often need to find ways to interrogate, summarize, modify and convert them. Perl is a highly efficient language for such tasks, and this module will focus on how the perl language can be used to accomplish these tasks.

We will first look at the basic structure of a perl program, and the specific syntax for this language. The course will start with an introduction to the basic programming concepts discussed in the previous module, as well as more esoteric perl-specific concepts. We will discuss the differences between scalar, array, and hash variables, and will also delve into the concept of variable referencing. We will discuss the concept of streams and file handles for reading and writing data, and how to deal with different types of data that you may encounter. We will look at how to install new perl modules to add new functionality depending on your goals. We will look at the idea of scripting, and creating programs that can be run with user input to achieve a specific task (e.g. writing a script to pull out a certain number of sequences from a fastq file). We will discuss ways to ensure that our code is efficient for dealing with large amounts of data.

We will also go into some detail about the concept of regular expressions for pattern recognition, which can be incredibly important concept in dealing with DNA, RNA and protein sequences. We will finally discuss using perl in real world situations, playing with some example data files to get a feel for how these concepts will work in practice.

3) Data analysis with R

R is an open source statistical language, used in many data sciences for statistical analyses. R libraries cover most statistical algorithms and allow for plotting of publication-quality figures. It is very well documented with a simple syntax, and is an excellent tool for data analysis.

In this module, we will first begin to understand the basic syntax of R, and how to load in and analyse data. In particular, the data frame class and how to interrogate these for data analysis will be considered.

We will also cover some statistics in this module, discussing the role of hypothesis testing in science and how we can go about setting up models to test our theories. Starting with basic summary statistics (such as mean, median, standard deviation, etc), we will eventually delve into the concept of linear models, and how these can be used to test association of potential explanatory variables with the dependent variable. We will look at how this relates to correlation analysis, move into t-test and ANOVA tests for comparing means between groups, non-parametric tests and when to use them, and eventually into more complicated multi-variable models and model selection methods.

We will use example data sets to explore R and some of the powerful analysis techniques that can be used to explore the data.

4) Solving biological problems with R

This course will extend the previous R course to look specifically at some of the ways that R can be used in biological data analyses. In particular, this course will look at some of the packages available through the Bioconductor project for accomplishing specific tasks that Bioinformaticians frequently come across.

First we will look at some of the more important biological classes in R. For instance, the GRanges class which extends the data range class for defining genomic regions. Understanding how GRanges can be compared and contrasted can make future analysis of any kind of genomic regions in R much easier and more efficient.

We will look at some example RNA seq data, and discuss ways of accessing gene annotation from Biomart and Ensembl, then summarising read level data on a per gene basis. With this summarised data, we will use packages limma, DESeq2 and edgeR to look for differentially expressed genes. We will then explore the various annotation packages available for converting between different accessions (Refseq, Ensembl, Hugo, etc), and move on to the use of annotation packages like GO.db for gene ontology analyses.

Next we will look at sequence packages such as the BioStrings genome packages, and ways to search for sequences to identify motifs, and compare known motifs and consensus sequences against a set of targets.

We will also look at typical visualisation approaches for biological data, particularly sequencing data. For instance, for ChIP seq of transcription factors, we may wish to look at how the binding looks at the TSS of known gene. For this, we can count reads in windows around known TSS, and use heatmaps to visualise the enrichment.

In addition we will look at ways to import and export data using the rtracklayer package, and how to port track data for visualisation using a genome browser (such as UCSC or IGV).

Finally we will touch upon the use of R Markdown, a way of creating dynamic documents with embedded R analyses which can be used to generate reports. Using knitr in R, R Markdown files containing embedded R code can be converted to documents of various formats (pdf, Microsoft Word, slideshow presentations, HTML web pages, and even interactive web applications using the shiny library) containing data, figures and tables generated on the fly. In particular, the idea of using R Markdown as a form of lab book will be suggested so that analyses are well documented.

5) Version Control

One important aspect of programming is version control, a method of managing and logging all changes made to programs in order to track the evolution of the code. Often a piece of code will serve a single purpose, will be used for a particular task, and then never consulted again. However, sometimes you will find yourself reusing, and more importantly changing code for future use. It is important to track and monitor these changes, particularly if you find that you ultimately break your code in unexpected ways.

Version control systems such as Git allow you to track changes as you make them, create branches of your working tree that can later be merged to allow testing of new ideas before committing to them, and tag your work at specific times (e.g. for incremental releases). Git also allows multiple users to work on a single project and allows all users to keep their code in sync with one another, which can be essential when working collaboratively with other Bioinformaticians.

This course will run through some of the basics of Git version control, how to initialise a repository and track changes as you go, how to create, merge and delete multiple branches, and how to fork repositories for collaborative coding.
